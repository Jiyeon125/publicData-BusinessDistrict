### 2. 데이터 결합 및 클린징 절차 (분석 프로세스 관점)

**1) 키 정의와 조인**

- 매출과 점포는 `기준_년분기_코드`, `행정동_코드_명`에 대응하는 동일 식별자, `서비스_업종_코드_명`까지 포함한 5개 키로 내부 조인합니다. 두 소스 모두에 존재하는 조합만 남겨 매출과 점포 지표가 항상 쌍을 이루도록 합니다.
- 인구는 분기 코드 + 행정동 수준만 맞춰 left 조인합니다. 특정 행정동 인구 레코드가 없으면 유동인구·인구당 매출 계산은 결측이 될 수 있어, 결과 해석 시 인구 데이터 공백 가능성을 염두에 둡니다.

**2) 스키마 정규화(컬럼 표준화)**

원본 파일의 한글 컬럼명·순서를 코드에서 표준 영문 변수명으로 매핑합니다(매출 최소 7열, 점포 최소 10열, 인구 최소 4열 등 위치 기준 검증 포함). 시간대별·요일별 매출 원본 컬럼명(예: `시간대_11~14_매출_금액`, `금요일_매출_금액`)이 상위 매출 테이블에 붙어 있으면 추가로 숫자형 변환합니다.  
점포 테이블에 프랜차이즈 점포 수 열이 12열 이상으로 존재하면 프랜차이즈 비율(프랜차이즈 점포 수 ÷ 점포 수)까지 파생합니다.

**3) 형 변환 및 이상치 대응**

- 문자열로 들어온 숫자(쉼표, 공백, `%` 포함)를 제거 후 `to_numeric`(오류 시 NaN) 처리합니다.
- 빈 문자열, `-`, `None` 등은 NaN으로 통일합니다.
- 분모가 0인 비율·비율형 지표는 0을 NaN으로 치환한 뒤 나눗셈하여 무한대(division by zero)를 방지하고, 결과의 `±inf`는 NaN으로 정리합니다.

**4) 행 단위 파생 변수**

결합 행 각각에 대해 예를 들어 다음을 계산합니다.

- 점포당 매출 금액·건수, 유동인구당 매출 금액·건수  
- 객단가(매출 금액 ÷ 매출 건수)  

**5) 창업유망 관련 지표**

필터가 적용되기 전 전체 마스터 데이터프레임 단위에서, 점포당 매출·유동인구·유동인구당 매출·폐업률(낮을수록 유리하도록 역변환한 안정성 점수)에 대해 0~100 min-max 정규화를 한 뒤, 가중합으로 행 단위 창업유망점수를 만듭니다. 대시보드에서는 사용자가 선택한 분기·업종·최소 점포 수 조건으로 행을 자른 다음, 행정동 이름(`dong_name`)별로 매출 합계·평균 및 위 점수들의 평균을 집계해 차트와 표에 반영합니다.  


### 3. 시각화 결과: 차트별 설명 및 인사이트

모든 차트는 사이드바 필터(분기, 업종, 최소 점포 수, Top N, 산점도 색 기준 등) 하에서의 부분 표본입니다. KPI와 순위 해석 시 현재 선택한 조건에서의 비교라는 전제를 붙입니다.


**실데이터 시나리오:** `기준_년분기_코드 = 20241`, 업종 `일식음식점`, 최소 점포 수 10, Top N 15 조건에서 대시보드 캡처를 바탕으로 정리한 인사이트는 [`insights_20241Q1_일식음식점.md`](./insights_20241Q1_일식음식점.md)를 참고해 주세요.

**1) 상단 KPI (4개 카드)**

- 총매출, 총 점포 수, 평균 점포당 매출, 평균 폐업률은 필터 구간 전체를 한 번에 요약합니다.  
- 인사이트 예시: 총매출은 규모의 큰 그림, 점포당 매출은 같은 점포 수 대비 매출 효율, 평균 폐업률은 상권 변동·경쟁 압력의 거친 신호로 읽을 수 있습니다(업종·분기 혼합 평균이므로 세부 해석은 동·업종별로 내려가야 함).

**2) 행정동별 총매출 Top N (가로 막대)**

- 필터 조건에서 총매출이 큰 행정동 순으로 정렬합니다. 툴팁에 총매출, 점포 수, 점포당 매출, 폐업률을 묶어 규모·효율·리스크를 동시에 볼 수 있게 합니다.  
- 인사이트: 1위 동은 조건 내 단일 최대 매출 규모를 의미할 뿐, 반드시 점포당 매출 또는 창업 적합성이 최고는 아닙니다. 대형 허브형 상권과 중소 규모 고효율 상권을 다음 차트와 함께 구분해서 읽는 것이 타당합니다.

**(3) 행정동별 점포당 매출 Top N (가로 막대)**

- 점포 1개당 매출이 높은 동을 보여 줍니다.  
- 인사이트: 상대적으로 적은 점포로 높은 매출을 내는 동은 매출 회전이나 업종 특성 상 단위 입지 효율이 높게 나타날 수 있습니다. 반면 총매출 순위와 점포당 매출 순위가 크게 어긋나는 동은 '규모는 작지만 호환이 좋다'거나 '규모만 크고 점포당으로는 평균 이하' 같은 패턴 진단에 유용합니다.

**(4) 유동인구와 매출 산점도**

- 축은 유동인구(합계)와 총매출, 점 크기는 점포 수, 색은 **폐업률 또는 창업유망점수 중 선택합니다.  
- 세로·가로 점선은 각각 유동인구·총매출의 평균으로, 화면을 네 사분면으로 나눕니다.  
- 인사이트(사분면 해석):
  - 유동인구↑·매출↑: 대형 활성 상권  
  - 유동인구↑·매출↓: 유동은 많으나 매출 전환은 약한 구간(경쟁 과다, 업종 부적합, 데이터 시차 등 검토)  
  - 유동인구↓·매출↑: 목적형·고전환 가능성(방문 목적이 뚜렷한 소비)  
  - 유동인구↓·매출↓: 저활성 상권  
  색으로 폐업률을 켜면 '사람은 많은데 위험도가 높은지'를 한 화면에서 대비할 수 있습니다.

**(5) 창업유망점수 Top N 표·막대그래프**

- 점수는 점포당 매출, 유동인구, 유동인구당 매출, 폐업률 안정성을 가중한 규칙 기반 복합지표입니다.  
- 표에는 동별 점수 구성요소와 실측 요약(총매출, 점포당 매출, 인구당 매출, 점포 수, 폐업률, 추천 문구)이 함께 나옵니다.  
- 인사이트: 순위 상위 동은 “한 지표만 튀는 곳”이 아니라 네 요소가 균형 있게 높거나, 폐업 리스크가 상대적으로 낮은 패턴일 때 유리합니다. 다만 가중치(0.35 / 0.25 / 0.25 / 0.15)는 업종·브랜드 전략에 맞게 조정 가능한 가정이므로, 필요 시 해당 프로젝트에서 가중치를 수정하여 사용할 수 있습니다.

**(6) Top 5 추천 카드**

- 상위 5개 행정동에 대해 창업유망점수, 점포당 매출·유동인구·인구당 매출·폐업률을 전체 평균과 병치하고, 규칙 기반으로 강점·주의점 불릿을 생성합니다.  
- 인사이트: 경영진·기획자에게 요약 페이지 역할을 하며, 평균 대비 무엇이 나은지, 무엇을 경계할지를 빠르게 전달합니다. 점포 수가 평균 대비 과다하면 카드 주의항목에 경쟁 강도를 짚도록 되어 있습니다.

**(7) 창업 후보지 상세 분석**

- 한 행정동을 선택해 핵심 지표, 동일 필터 안 전체 평균 대비 표, 업종 필터 적용 상태에서의 순위(총매출, 점포당 매출, 유동인구, 인구당 매출, 폐업 안정성, 창업점수 등), 강점·주의, 종합 문구(긍정/혼재/신중 검토)를 제공합니다.  
- 인사이트: Top N 차트에서 '왜 상위인가'를 내려다보려면 이 섹션이 필요합니다. 순위는 현재 선택 업종 안에서만 의미 있으므로, 업종을 바꿀 때마다 순위가 달라짐을 주의하여야 합니다.

**(8) 시간대별·요일별 매출 비중 (데이터에 컬럼이 있을 때만)**

- 선택한 행정동·필터 내 모든 행에 대해 시간대별·요일별 매출 금액을 합산한 뒤 막대그래프로 보여 줍니다.  
- 인사이트: 피크 시간대(예: 점심·저녁·심야)와 피크 요일(평일 대비 주말 등)에 따라 영업 시간·인력·프로모션 설계를 논의할 수 있습니다. 컬럼이 없으면 해당 그래프는 생략됩니다.

## 프로젝트 개요

서울시 상권분석서비스 2024년 행정동 단위 데이터를 **Supabase**에서 불러와 분석하는 Streamlit 대시보드입니다. 행정동별 매출·점포·유동인구를 결합하고, 규칙 기반 **창업유망점수**를 산출하여 후보 지역을 탐색할 수 있습니다. Gemini API는 포함하지 않습니다.

### 주요 기능

- 분기·업종·최소 점포 수·Top N 등 **사이드바 필터**
- KPI 카드: 총매출, 총 점포 수, 평균 점포당 매출, 평균 폐업률
- 행정동별 **총매출** / **점포당 매출** Top N 막대그래프 (간결한 툴팁)
- 유동인구 대비 매출 **산점도** + 평균선(점선) 및 사분면 해석 안내
- **창업유망지역 Top N** 표·막대그래프, Top 5 **추천 카드**(강점·주의점, 전체 평균 대비 표시)
- **창업 후보지 상세 분석**: 행정동 선택, 핵심 지표, 평균 대비 표, 업종 내 순위, 강점·주의점, 종합 판단 문구
- **시간대별·요일별 매출 비중** 그래프 및 자동 해석 문구(해당 컬럼이 있을 때만 표시)
- 금액·건수·비율·점수에 대한 **일관된 포맷**(억/만 원, 콤마, % 등)

```bash
# macOS / Linux
source .venv/bin/activate
```

3. 의존성 설치:

```bash
python -m pip install -r requirements.txt
```

4. Supabase 접속 정보를 Streamlit secrets에 둡니다.

**파일 위치:** `.streamlit/secrets.toml` (Git에는 올리지 마세요.)

```toml
[supabase]
url = "https://YOUR_PROJECT_ID.supabase.co"
key = "YOUR_SUPABASE_ANON_KEY"
```

아래처럼 **최상위 키**만 두어도 `app.py`에서 인식합니다.

```toml
SUPABASE_URL = "https://YOUR_PROJECT_ID.supabase.co"
SUPABASE_KEY = "YOUR_SUPABASE_ANON_KEY"
```

또는 `connections.supabase` 블록의 `url`/`key` 또는 `SUPABASE_URL`/`SUPABASE_KEY` 조합도 지원합니다.

5. 앱 실행:

```bash
python -m streamlit run app.py
```

브라우저에서 표시되는 주소(기본 `http://localhost:8501`)로 접속합니다.

> Supabase 조회 실패 시 로컬 CSV로 넘어가지 않습니다. secrets·네트워크·테이블 권한·RLS 설정을 확인하세요.

## Streamlit Cloud 배포

1. GitHub에 코드를 푸시합니다 (`secrets.toml`·원본 CSV는 커밋하지 않음).

2. [Streamlit Community Cloud](https://streamlit.io/cloud)에서 저장소와 브랜치를 연결하고, **앱 설정 → Secrets**에 동일한 Supabase URL·키를 TOML 형식으로 입력합니다.

3. 메인 파일을 `app.py`로 지정합니다.

4. 첫 실행 시 세 테이블 전량을 페이지 단위로 가져오므로 데이터 크기에 따라 로딩이 길어질 수 있습니다. `@st.cache_data(ttl=300)`으로 짧은 기간 캐시됩니다.

## 파생 지표

결합된 행 단위에서 다음과 같이 계산합니다(0으로 나누는 경우 NaN 처리).

- **점포당매출** = 당월 매출 금액 ÷ 점포 수  
- **점포당매출건수** = 당월 매출 건수 ÷ 점포 수  
- **유동인구당매출** = 당월 매출 금액 ÷ 총 유동인구 수  
- **유동인구당매출건수** = 당월 매출 건수 ÷ 총 유동인구 수  
- **객단가** = 당월 매출 금액 ÷ 당월 매출 건수  
- **프랜차이즈비율** = 프랜차이즈 점포 수 ÷ 점포 수 (해당 컬럼이 있을 때)

집계 화면에서는 행정동별 합계·평균 등으로 요약해 KPI·차트에 사용합니다.

## 창업유망점수 계산식

각 구성 지표를 해당 필터 구간 전체에 대해 **0~100 min-max 정규화**한 뒤 가중합합니다.

```text
창업유망점수 =
  0.35 × 점포당매출점수
+ 0.25 × 유동인구점수
+ 0.25 × 유동인구당매출점수
+ 0.15 × 폐업률안정성점수
```

- **폐업률안정성점수** = 100 − (폐업률에 대한 min-max 점수)  
- 규칙 기반 추천 문장은 위 세부 점수·평균 대비 등과 연동되어 카드·상세 분석에 표시됩니다.

## 툴팁·표시 규칙 (요약)

- Plotly 그래프는 `custom_data`와 `hovertemplate`으로 항목 수를 줄이고 한글 라벨을 사용합니다.  
- 금액은 **억 원 / 만 원 / 원** 단위로 읽기 쉽게 표시합니다.  
- 비율은 **%**, 점수는 소수 **한 자리** 정도로 통일합니다.  
- `<extra></extra>`로 트레이스 이름 등 불필요한 hover 줄을 숨깁니다.

## GitHub에 원본 CSV와 secrets를 올리지 않는 이유

- 원본 CSV 용량이 크고 버전 교체가 잦을 수 있어 저장소와 분리하는 편이 관리에 유리합니다.  
- Supabase URL·API 키는 공개 저장소에 포함하면 안 됩니다.  
- `.gitignore`에서 `data/*.csv`, `.streamlit/secrets.toml`, `.env` 등을 제외하고 `data/.gitkeep`만 추적할 수 있습니다.

## 데이터 한계 및 유의사항

- 행정동·업종·분기 단위 **공개 통계** 기준이며, 실제 매장 단위 매출이 아닙니다.  
- **임대료, 권리금, 실제 영업이익, 입지 세부 조건**은 반영하지 않습니다.  
- 창업유망점수와 추천 문구는 **참고용 탐색 도구**이며, 최종 의사결정 전 현장 조사·추가 분석이 필요합니다.
